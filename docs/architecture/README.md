# Архітектура фронтенду

У репозиторії підтримуються два незалежні фронтенд-застосунки. Їхні правила,
рішення та SDD-документи не змішуються, оскільки застосунки використовують різні
фреймворки й підходи до стану.

- [Admin (React)](admin/README.md) — архітектура застосунку `apps/admin`.
- [Storefront (Angular)](storefront/README.md) — архітектура застосунку
  `apps/storefront`.

Загальні для репозиторію рішення описуються лише тоді, коли вони дійсно однаково
застосовні до обох застосунків. Інакше ADR та стандарти розміщуються в розділі
відповідного застосунку.
