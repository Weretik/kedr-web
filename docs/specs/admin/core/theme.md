# Корпоративная тема Admin

## Цель

Как пользователь Admin, я хочу использовать светлую, тёмную или системную цветовую схему, чтобы интерфейс соответствовал корпоративной палитре и моим настройкам отображения.

## Границы

- In scope: корпоративная палитра MUI для `apps/admin`, светлая и тёмная схемы, выбор светлой, тёмной или системной схемы в верхней панели, сохранение выбора средствами MUI.
- Out of scope: изменение Angular storefront, разработка новых дизайн-токенов, синхронизация предпочтений с серверным профилем пользователя.
- Затрагиваемые библиотеки: `apps/admin`, `admin/core/shell`.

## Сценарии

1. Given пользователь впервые открывает Admin, When в браузере задана системная тема, Then интерфейс использует соответствующую светлую или тёмную схему.
2. Given пользователь выбирает светлую или тёмную тему, When он обновляет страницу или открывает другую вкладку Admin, Then MUI восстанавливает выбранный режим.
3. Given пользователь выбрал системную тему, When меняется системная настройка цвета, Then Admin следует новой системной схеме.

## Контракты и данные

- HTTP endpoint: отсутствует.
- Состояние: режим `light`, `dark` или `system` хранится стандартным storage manager MUI в `localStorage`.
- Палитра: источником значений служит `apps/storefront/src/styles/generated.tailwind-palette.css`.
- Переключение: `colorSchemeSelector: 'data'` обязателен, так как MUI с двумя схемами по умолчанию использует `media`, который не поддерживает ручное переключение.
- `secondary` в тёмной схеме использует тёмные нейтральные оттенки, чтобы шапка и её контролы сохраняли контраст.

## UI и маршрутизация

- Route: все маршруты Admin.
- Компоненты: MUI `ThemeProvider`, `CssBaseline`, селектор режима в `AdminLayout`.

## Критерии приёмки

- [ ] MUI использует корпоративные primary, neutral, info, success, warning и danger цвета в обеих схемах.
- [ ] Пользователь может выбрать `System`, `Light` или `Dark` с клавиатуры и указательным устройством.
- [ ] Системный режим является режимом по умолчанию.
- [ ] Выбранный режим сохраняется между перезагрузками и синхронизируется вкладками средствами MUI.
- [ ] Шапка Admin использует цвет `secondary` в светлой и тёмной схемах.
- [ ] Селектор темы в шапке имеет контрастные текст, метку, иконку и рамку в обеих схемах.

## Проверка

- Unit: не требуется для декларативной MUI-конфигурации.
- Integration / e2e: не требуется.
- Ручная проверка: изменить системный режим и проверить все три значения селектора.
