# SDD: міграція шаблону Admin — фаза 0, фіксація референсу

## Мета

Зафіксувати межі перенесення візуального шаблону без реалізації інтерфейсу.

## Референс

- [Локальний шаблон](file:///D:/RiderProjects/template) — єдине джерело структури, розмірів, станів і візуальних рішень.

## Межі та рішення

- Референсом є лише структура, типографіка, відступи, elevation і поведінка компонентів; фірмові кольори не переносяться.
- Підтверджені маршрути: dashboard і products; customers, settings та account потребують окремих контрактів.
- У межах майбутніх фаз: тема, shell, навігація, загальні візуальні примітиви та dashboard.
- Поза межами: вигадані дані, API-контракти відсутніх доменів і промо-блоки шаблону.

## Критерії приймання

- Для кожної наступної фази визначено scope та залежності.
- Скріншоти не є вимогою фази.
- Відкриті питання зафіксовані в головній специфікації міграції.
