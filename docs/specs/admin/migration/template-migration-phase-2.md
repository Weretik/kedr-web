# SDD: міграція шаблону Admin — фаза 2, shell і навігація

## Мета

Реалізувати адаптивний каркас Admin: sidebar, верхню панель, mobile drawer і
навігацію до наявних маршрутів.

## Межі

- `admin/core/shell` містить layout, навігацію, меню профілю, сповіщень і теми.
- Sidebar використовує secondary-колір, divider, логотип і компактні пункти меню.
- Верхня панель містить сповіщення, перемикач теми поруч із меню менеджера та avatar менеджера.
- Усі видимі користувачу тексти — українською.
- Дані профілю й logout надходять лише через наявний session provider; API не додається.

## Сценарії

1. На desktop користувач бачить sidebar та активний маршрут.
2. На mobile відкриває drawer, вибирає маршрут, після чого drawer закривається.
3. Користувач змінює світлу, темну або системну тему.
4. Користувач відкриває меню менеджера й завершує сесію.

## Критерії приймання

- Навігація доступна з клавіатури та має видимий focus.
- Shell не змінює публічний API `AdminLayout`.
- Мобільна й desktop-навігація мають однакові маршрути та active-стан.
- Icon-only кнопки мають доступні назви.

## Перевірка

- `npx nx lint admin`
- `npx nx build admin`
- Ручна перевірка на `320px`, `768px`, `1200px` і в усіх схемах кольорів.
