# SDD: міграція шаблону Admin — фаза 1a, структура теми

## Мета

Розділити тему на логічні модулі без зміни її зовнішнього контракту.

## Структура

- `admin-theme.ts` — композиція теми.
- `palette.ts`, `typography.ts`, `shadows.ts` — токени.
- `components.ts` — component overrides.

## Критерії приймання

- Імпорти споживачів теми не змінюються.
- Кожен файл має одну відповідальність.
- Стилі не дублюються локально, якщо їх можна задати темою.

## Перевірка

- `npx nx lint admin`
- `npx nx build admin`
