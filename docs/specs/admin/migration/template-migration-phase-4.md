# SDD: міграція шаблону Admin — фаза 4, dashboard

**Статус:** виконано 2026-07-23.

## Мета

Замінити черновий dashboard на композицію за структурою референсного шаблону
зі статичними прикладами даних для візуальної перевірки.

## Контракти та дані

- Dashboard data-access і API-контракти метрик відсутні.
- До появи контрактів dashboard показує статичні приклади даних для візуальної
  перевірки композиції.
- Після погодження контракту дані надходять лише через окрему dashboard
  data-access library; компонент сторінки не виконує HTTP-запити напряму.

## Межі

- Dashboard має заголовок, чотири statistic cards, bar chart продажів, donut
  chart джерел трафіку, список товарів і таблицю останніх замовлень.
- Приклади значень не пов’язані з API та будуть замінені data-access після
  погодження контракту.

## Поза межами

- Запити до API та створення dashboard data-access до погодження контракту.
- Виведення версії застосунку, адреси API чи HTTP-логів у користувацькому UI.
- Створення dashboard data-access до погодження API-контракту.

## Сценарії

1. Користувач відкриває `/dashboard` і бачить структурований демонстраційний
   огляд: statistic cards, графік продажів і останні оновлення.
2. Після погодження API-контракту статичні приклади замінюються даними через
   dashboard data-access.
3. На ширинах 320px, 768px і 1200px картки та панелі залишаються читабельними.

## Критерії приймання

- Немає Devias-текстів або технічних параметрів серед користувацького контенту.
- Dashboard використовує `StatisticCard` з фази 3.
- Усі повідомлення користувачу українською.
- Сторінка не має прямих HTTP-викликів.

## Перевірка

- Unit test `DashboardPage` перевіряє заголовок, графіки та списки dashboard.
- `npx nx lint admin-dashboard-feature`.
- `npx nx build admin`.
- Ручна перевірка на 320px, 768px і 1200px у світлій і темній схемах.

## Результат

- Реалізовано dashboard за композицією референсу: чотири statistic cards,
  MUI X Charts bar/donut, останні товари та останні замовлення.
- Значення є статичними прикладами та не пов’язані з API; після погодження
  контракту їх замінює dashboard data-access.
- Пройдено `npx nx test admin-dashboard-feature`, lint і `npx nx build admin`.
- Ручна візуальна перевірка не виконана: вбудований браузер був недоступний у
  середовищі виконання.
