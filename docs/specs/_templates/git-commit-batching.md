# SDD: поділ поточних змін на коміти

## Мета

Взяти всі незакомічені зміни поточного робочого дерева та створити кілька
невеликих логічно пов’язаних комітів у наявному стилі репозиторію.

## Межі

- Включити лише зміни, які користувач доручив закомітити.
- Не змінювати вміст файлів без потреби для перевірки або виправлення явної помилки форматування.
- Не змішувати незалежні області: конфігурацію, документацію, інфраструктуру й окремі можливості.
- Не створювати commit, push, branch або PR без явного доручення користувача.

## Порядок роботи

1. Показати `git status --short`, staged/unstaged diff і останні коміти.
2. Скласти групи змін із зрозумілою метою кожного коміту.
3. Виконати доступні точкові перевірки змінених областей.
4. Додавати файли до index окремими групами та створювати окремі commit.
5. Використовувати стиль повідомлень із `git log`, зокрема Conventional Commits.
6. Перевірити `git status --short`, `git diff --check` та підсумкову історію.

## Критерії приймання

- [ ] Робоче дерево чисте або залишені зміни явно зазначені як виключення.
- [ ] Кожен коміт має одну логічну мету та зрозуміле повідомлення.
- [ ] Незалежні зміни не об’єднані в один великий коміт.
- [ ] У звіті вказані результати та невиконані перевірки.
