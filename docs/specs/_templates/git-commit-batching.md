# SDD: разбиение текущих изменений на коммиты

## Цель

Взять все незакоммиченные изменения текущего рабочего дерева и создать несколько небольших, логически связанных коммитов в существующем стиле репозитория.

## Входные данные

- Текущая ветка и `git status`.
- Недавняя история: `git log --oneline`.
- Staged, unstaged и untracked изменения.

## Границы

- Включить все изменения, которые пользователь поручил закоммитить.
- Не изменять содержимое файлов без необходимости для проверки или исправления явной ошибки форматирования.
- Не смешивать независимые области: конфигурация, документация, инфраструктура и отдельные пользовательские возможности должны быть в разных коммитах.
- Не создавать commit, push, branch или PR без явного поручения пользователя.

## Порядок работы

1. Показать `git status --short`, staged/unstaged diff и последние коммиты.
2. Составить группы изменений с понятной причиной для каждого коммита.
3. Выполнить доступные точечные проверки для изменённых областей.
4. Добавлять файлы в index по одной группе и создавать отдельный commit.
5. Использовать стиль сообщений из `git log`; для этого репозитория — Conventional Commits с scope, например `feat(admin): ...`.
6. Проверить `git status --short`, `git diff --check` и итоговую историю.

## Критерии приёмки

- [ ] Рабочее дерево чистое или оставшиеся изменения явно перечислены как исключённые.
- [ ] Каждый коммит имеет одну логическую цель и понятное сообщение.
- [ ] Нет одного большого смешанного коммита при наличии независимых изменений.
- [ ] Результаты проверок и невыполненные проверки указаны в отчёте.

## Готовый запрос

> Возьми все незакоммиченные изменения в текущем рабочем дереве. Проанализируй diff и недавнюю историю, раздели изменения на несколько логически связанных коммитов в стиле этого репозитория, выполни доступные точечные проверки и создай коммиты. Не делай push и не меняй содержимое без необходимости. В конце покажи хеши коммитов, результаты проверок и оставшиеся изменения.
